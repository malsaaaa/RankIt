# RankeRating

**RankeRating** is a Mobile Cloud Computing (MCC) mobile application built with Flutter. It allows users to create custom ranking categories, add candidate items, cast votes via an interactive drag-and-drop interface, view live-aggregated leaderboards, and obtain intelligent pop-culture summaries generated by Gemini AI.

---

## 🚀 Key Features

1.  **User Authentication**: Firebase Authentication (Email/Password) with persistent session checks and profile management.
2.  **Category Creation**: Organise lists into Food, Games, Movies, Tourist Attractions, Technology, etc.
3.  **Dynamic Lists & Candidates**: Add list topics under any category, along with candidate items containing names, descriptions, and pictures.
4.  **Drag-and-Drop Ballots**: Drag items up and down to arrange your Top 10 choice.
5.  **Borda Count Aggregation**: Votes automatically update community leaderboards (Rank 1 gets 10 points, Rank 2 gets 9 points, ..., Rank 10 gets 1 point).
6.  **Real-Time Standings**: Community scoreboard positions animates live as votes are cast using reactive streams.
7.  **AI Analytical Summaries**: Integrates Gemini API (`gemini-1.5-flash`) to generate automated insights on community preferences.
8.  **Cloud Image Storage**: Pick cover images and upload them directly to Cloudinary.
9.  **Full Search & Filters**: Search categories and topics on-the-fly.
10. **Resilient Local Fallback**: Swaps to in-memory local databases, mock auth, and Unsplash templates automatically if credentials or connections are missing.

---

## 🛠️ Architecture & Tech Stack

*   **Front-End**: Flutter (Dart)
*   **State Management**: Provider (ChangeNotifier)
*   **Theme**: Modern dark Obsidian base with glassmorphic cards and glowing highlights
*   **Cloud Integrations**:
    *   Firebase Authentication
    *   Firebase Firestore (Real-time scoreboard streams)
    *   Cloudinary (Category & item cover photos)
    *   Gemini API (generative pop-culture summaries)
*   **Backend Interface Layer**: Designed with clean repository abstractions. The application references a base interface service that currently binds to Firestore, allowing you to easily swap the implementation with a **Laravel REST API & MySQL Database** backend.

## 🚀 Setup and Installation

### 1. Pre-requisites
Ensure you have the Flutter SDK installed on your system.

```bash
flutter --version
```

### 2. Install Dependencies
Run the command below in the project root:

```bash
flutter pub get
```

### 3. Setup Secrets & API Credentials
Open the [secrets.dart](file:///c:/Users/HP%20VICTUS/AndroidStudioProjects/rankerating/lib/config/secrets.dart) file (`lib/config/secrets.dart`) and populate your API details:

```dart
class Secrets {
  static const String geminiApiKey = 'YOUR_GEMINI_API_KEY';
  static const String cloudinaryCloudName = 'YOUR_CLOUDINARY_CLOUD_NAME';
  static const String cloudinaryUploadPreset = 'YOUR_CLOUDINARY_UPLOAD_PRESET';
}
```
*Note: You can also specify these values at compile-time using `--dark-define` environment flags.*

### 4. Running the Application
Launch the dev runner:

```bash
flutter run
```

### 5. Running Tests
Verify that all unit and widget tests pass:

```bash
flutter test
```

## 🔗 Connecting to Laravel REST API + MySQL Backend

The codebase contains a clean data isolation abstraction. All voting, listing, and user operations call functions declared in `BaseDatabaseService` and `BaseAuthService`. 

To connect to your Laravel REST API service instead of Firebase:
1. Create a class `LaravelDatabaseService` implementing `BaseDatabaseService` inside `lib/services/`.
2. Connect endpoints to your Laravel REST API endpoints (using the `http` package).
3. Update `lib/providers/ranking_provider.dart` to instantiate `LaravelDatabaseService` instead of `DatabaseService`.
